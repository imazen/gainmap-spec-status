# ITU-R BT.2446-1 — Methods for conversion of HDR content to SDR content and vice-versa

## Identification

- **Reference:** Report ITU-R BT.2446-1 (03/2021)
- **Title:** Methods for conversion of high dynamic range content to standard
  dynamic range content and vice-versa
- **Download:** https://www.itu.int/dms_pub/itu-r/opb/rep/R-REP-BT.2446-1-2021-PDF-E.pdf
- **Supersedes:** BT.2446-0 (04/2019)

## Why this matters for gain maps

BT.2408 §5.2 cites BT.2446 as the source for three HDR→SDR conversion
methods. When an encoder needs to produce an SDR base from an HDR master
for gain map encoding, or needs to produce an HDR alternate from an SDR
base, these are the reference methods the broadcast industry uses.

The document provides three matched TMO/ITMO pairs with different tradeoffs
(graded content vs live broadcast vs round-trip fidelity), plus a
psychophysical round-trip evaluation showing that Method A's degradation is
imperceptible in controlled conditions. This is directly relevant to the
gain map encode/decode cycle: the SDR base quality determines the gain map's
effectiveness, and round-trip stability matters for re-encode workflows.

## Structure

- **§2** — Guidelines for SDR↔HDR conversion (requirements, pitfalls)
- **§3** — Feature comparison table of Methods A, B, C
- **§4** — Method A: graded content (1000→100 cd/m2 and inverse)
- **§5** — Method B: live broadcast (robust to clipped SDR)
- **§6** — Method C: skin-tone-preserving with parametric adjustments
- **Annex 1** — Psychophysical evaluation of Method A round-trip

## Method A — graded content (§4)

### TMO: HDR (1000 cd/m2) → SDR (100 cd/m2)

The tone mapping operates in a perceptually linear space via a three-step
process:

**Step 1 — Nonlinear transfer + perceptual linearization:**

Input: normalized full-range linear display-light RGB in BT.2100 primaries
at 1000 cd/m2.

```
R' = R^(1/2.4)          // nonlinear transfer
G' = G^(1/2.4)
B' = B^(1/2.4)

Y' = 0.2627*R' + 0.6780*G' + 0.0593*B'    // luma

rho_HDR = 1 + 32 * (L_HDR / 10000)^2.4     // where L_HDR = 1000

Y'_p = log(1 + (rho_HDR - 1) * Y') / log(rho_HDR)   // perceptual linearization
```

**Step 2 — Piecewise tone curve (the knee):**

```
Y'_c = 1.0770 * Y'_p                                    for Y'_p <= 0.7399
Y'_c = -1.1510*Y'_p^2 + 2.7811*Y'_p - 0.6302            for 0.7399 < Y'_p < 0.9909
Y'_c = 0.5000*Y'_p + 0.5000                              for Y'_p >= 0.9909
```

**Step 3 — Convert back from perceptual to gamma domain:**

```
rho_SDR = 1 + 32 * (L_SDR / 10000)^2.4     // where L_SDR = 100

Y'_SDR = (rho_SDR^Y'_c - 1) / (rho_SDR - 1)
```

**Colour correction** (Table 3): Applied in Y'Cb'Cr' space to compensate
for Hunt effect (perception of colour changes with luminance):

```
f(Y'_SDR) = Y'_SDR / (1.1 * Y')    // colour scaling function

C'_b,TMO = f(Y'_SDR) * (B' - Y') / 1.8814
C'_r,TMO = f(Y'_SDR) * (R' - Y') / 1.4746

Y'_TMO = Y'_SDR - max(0.1 * C'_r,TMO, 0)     // adjusted luma
```

Output: Y'_TMO, C'_b,TMO, C'_r,TMO → convert to R'G'B' via BT.2020.

The entire process can be implemented as a single 1D LUT.

### ITMO: SDR (100 cd/m2) → HDR (1000 cd/m2) (§4.2)

The approximate inverse of the TMO above. Uses a similar perceptual
linearization scheme with different polynomial coefficients for the
expansion curve:

```
Y'_HDR = a1*Y''^2 + b1*Y'' + c1          for Y'' <= T
Y'_HDR = a2*Y''^2 + b2*Y'' + c2          for Y'' > T
```

Where Y'' = 255.0 * Y' (scaled to 0–255 range) and:

```
a1 = 1.8712e-5,  b1 = -2.7334e-3,  c1 = 1.3141
a2 = 2.8305e-6,  b2 = -7.4622e-4,  c2 = 1.2528
T = 70
```

Chroma is scaled by factor `S_C` to compensate for the Hunt effect:

```
S_C = Y / (Y^1.075)    if Y > Y'
S_C = 1                if Y <= Y'
```

The output is absolute HDR linear light at up to `L_max = 1000 cd/m2`,
suitable for PQ encoding or HLG encoding via BT.2408 §6.2.

### Round-trip quality (Annex 1)

Psychophysical evaluation with 12 participants (2AFC, 5-second viewing,
Sony BVM-X300, dark room, 115 HDR images):

- **HDR round-trip** (HDR → SDR → HDR): participants could not distinguish
  processed from original (p = 0.167, not significant).
- **SDR round-trip** (SDR → HDR → SDR): participants could not distinguish
  processed from original (p = 0.196, not significant).
- Even triple-conversion (HDR → SDR → HDR → SDR) showed no detectable
  degradation.

**Conclusion:** Method A's TMO/ITMO pair produces imperceptible round-trip
losses under controlled conditions. This is the strongest round-trip
evidence of the three methods.

## Method B — live broadcast (§5)

Designed for SDR content with large over-exposed/clipped areas (common in
live sport). Uses a more conservative highlight expansion.

### ITMO: SDR → HDR (§5.1)

1. **Linearize** SDR via BT.1886 EOTF at 100 cd/m2
2. **Convert** to CIE 1976 Yu'v' for luminance-only processing
3. **Scale and gamma-compensate:** double display light (100→200 cd/m2),
   apply OOTF gamma ~1.1 (from BT.2100 Note 5f extended formula)
4. **Inverse tone map highlights:** above a breakpoint of ~78% SDR signal
   (~55 cd/m2 on 100 cd/m2 display), apply exponential expansion. The
   gradient at the breakpoint is unity for smooth transition.
5. **Apply HLG or PQ inverse EOTF**

**Simplified HLG path (§5.1.5):** By setting HLG EOTF peak luminance
L_W = 291 cd/m2 with gamma ≈ 0.99–1.03, a 1:1 mapping of SDR luminance
to HDR luminance is achieved over most of the signal range without
separate scaling or gamma compensation. 90% SDR → 75% HLG (HDR Reference
White). This avoids the separate linearize-scale-compensate-encode chain.

The simplified path limits peak HDR luminance to 291 cd/m2 (vs Method A's
1000 cd/m2), which is deliberate: it prevents over-exposed SDR areas from
becoming objectionably bright highlights.

### TMO: HDR → SDR (§5.2)

Complementary to the ITMO, designed for minimal round-trip loss:

1. **Apply HLG EOTF** at L_W = 291 cd/m2 (gamma ≈ 1.03) to get linear
   display light. This maps HDR Reference White (75% HLG) to ~86-90%
   of the SDR signal.
2. **Convert** to CIE 1976 Yu'v'
3. **Tone map luminance only:** 1:1 below breakpoint (~78% SDR),
   logarithmic compression above (inverse of the exponential expansion).
4. **Hard clip out-of-gamut colours** to BT.709 colour volume by
   desaturation in ICTCP space (avoids hue shifts seen with Yu'v' or
   CIE L\*a\*b\*)
5. **Convert** to BT.709 RGB and apply inverse BT.1886 EOTF

### Round-trip

The TMO/ITMO pair achieves excellent round-trip below the 78% breakpoint
(exact 1:1 mapping). Above the breakpoint, the exponential expansion and
logarithmic compression slowly diverge, producing small highlight losses.
Peak luminance is slightly reduced after double conversion.

## Method C — skin-tone-preserving, parametric (§6)

A parametric method designed around preserving skin tone relationships
between HDR and SDR. The tone mapping curve has adjustable parameters
determined from skin tone analysis (Annex 4 of BT.2408).

### TMO: HDR → SDR (§6.1)

1. **Convert HLG to linear display light** (HLG EOTF, gamma 1.2)
2. **Crosstalk matrix** — reduces saturation to prevent hue shifts from
   subsequent clipping:
   ```
   [Rx]   [1-2α  α    α  ] [R]
   [Gx] = [α    1-2α  α  ] [G]
   [Bx]   [α     α   1-2α] [B]
   ```
   Where α ∈ [0, 0.33] controls desaturation degree.
3. **Convert to CIE 1931 Yxy** — separates luminance for tone mapping
4. **Tone map luminance** — piecewise linear + log:
   ```
   Y_SDR = k1 * Y_HDR                                       for Y_HDR < Y_HDR,ip
   Y_SDR = k2 * ln(Y_HDR / Y_HDR,ip) + k4                   for Y_HDR >= Y_HDR,ip
   ```
   Where the inflection point `Y_HDR,ip` corresponds to ~80% SDR
   (~58.5 cd/m2 on 100 cd/m2 display — chosen to sit above skin tones).

   Default parameters (from BT.2408 Annex 4 skin tone analysis, HDR ref
   white → 96% SDR):
   ```
   k1 = 0.83802,  k2 = 15.09968,  k3 = 0.74204,  k4 = 78.99439
   ```

   Parameters are derivable from: the linear gain k1 (set by skin tone
   relationship), the inflection point (set at 80% SDR), and the SDR
   white level for HDR Reference White. Equations (6)-(10) in the spec
   provide the constraints.

5. **Convert back to linear RGB** using preserved xy chromaticity
6. **Inverse crosstalk matrix** — restores original saturation
7. **Apply inverse BT.1886 EOTF** → SDR output

**Optional: chroma correction above HDR Reference White** (§6.1.8) —
progressively desaturates highlights toward achromatic in CIE L\*a\*b\*
space, controlled by parameter σ. At σ=1, HDR peak is fully white. This
matches the SDR production convention of representing highlights as
achromatic.

### ITMO: SDR → HDR (§6.2)

The mathematical inverse of the TMO (excluding the optional chroma
correction). Because the TMO uses a piecewise linear+log function with
closed-form inverse, the round-trip is exact when the optional processing
is excluded.

Key feature: HDR Reference White (75% HLG) maps to ~95% SDR signal level
(vs ~90% in Method B, ~100% in Method A). This uses SDR super-whites
(0–109% signal range) to preserve highlight headroom.

## Design guidelines (§2) — relevant to gain map encoding

The spec's guidelines for SDR→HDR conversion directly apply to choosing
how to produce an SDR base for gain map encoding:

1. **Maintain shadow detail.** Non-linear expansion; linear scaling
   raises mean luminance too high for comfortable viewing.
2. **Don't over-expand midtones.** They become uncomfortably bright and
   alter the scene's perceived look. Expansion should be concentrated
   in highlights.
3. **Expand highlights up to peak luminance** insofar as input quality
   allows. 8-bit content with JPEG compression limits safe expansion
   (echoes BT.2408 §5.5 and our banding analysis).
4. **Adjust chromatic content** for the Hunt effect — saturation
   perception changes with luminance. All three methods include color
   correction (Method A in Y'Cb'Cr', Method B bypasses, Method C in
   Yxy + optional L\*a\*b\*).
5. **Over-exposed areas** in SDR should not be naively expanded to high
   luminance levels. De-clipping or containment in the SDR range is
   preferred.
6. **Temporal stability** — no method should introduce temporal artifacts.
   All three methods are static (per-frame, no temporal adaptation),
   which makes them safe but potentially suboptimal for varying scenes.
   Dynamic methods exist but are computationally demanding.

## Implications for zentone and gain map encoding

1. **Method A is the reference for graded content.** Its TMO produces
   the SDR base quality that gain map encoders should target when
   working with high-quality HDR masters. The perceptually verified
   round-trip means gain map decode → re-encode cycles will be stable.

2. **Method B is the reference for live/broadcast content.** Its
   conservative 291 cd/m2 ceiling and simplified HLG path align with
   the "hybrid-linear" workflow from BT.2408 Annex 10 (NBCU). The
   gain map's declared `max_content_boost` should match the TMO's
   expansion ceiling.

3. **Method C's parametric design** is closest to what an adaptive
   gain map encoder would want — the ability to tune the tone curve
   based on content statistics (skin tones, highlight distribution).
   zentone's `AdaptiveTonemapper` is the zen equivalent.

4. **All three methods warn about 8-bit banding.** This reinforces the
   banding analysis in `apply-math-and-banding.md`: the SDR base's
   quantization artifacts are amplified by the gain map multiply,
   and the TMO's highlight expansion amplifies them further.

5. **The colour correction steps** (Hunt effect compensation) are
   important for gain map quality — a gain map computed from an SDR
   base that doesn't account for the Hunt effect will produce
   incorrect colours when reconstructed at high luminance. zentone
   should ensure its SDR base generation includes Hunt-effect-aware
   chroma scaling.

6. **Method C's exact inverse** is the strongest round-trip guarantee.
   For gain map re-encode workflows where exact SDR↔HDR round-trip
   matters (e.g., imageflow processing a UltraHDR JPEG), detecting
   that the original used Method C's parametric curve enables
   perfect reconstruction.
